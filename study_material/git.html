Introduction to Git & Version Control

What is Version Control?

System that records changes to files over time. Enables rollback, collaboration, and tracking.

Centralized vs Distributed VCS

Centralized (SVN)Distributed (Git)
Single serverFull local copy
Network requiredOffline work
SlowerFast branching

What is Git?

Fast, distributed VCS created by Linus Torvalds. Uses snapshots, not deltas.

GitHub / GitLab / Bitbucket

  • Remote hosting
  • Pull Requests
  • CI/CD
  • Issue tracking

Core Git Concepts

Repository Structure

  • Working Directory: Your files
  • Staging Area: git add
  • Commit History: .git folder

Basic Commands

git init
git clone https://github.com/user/repo.git
git add .
git commit -m "feat: add login"
git push origin main

Branching & Merging

Branches

git branch feature/auth
git checkout feature/auth
git checkout -b feature/payment  # create + switch

Merge Types

  • Fast-forward: No divergence
  • 3-way merge: Creates merge commit

Merge Conflict

<<<<<<< HEAD
const title = "Home";
=======
const title = "Dashboard";
>>>>>>> feature/login

Undoing & Reverting Changes

git restore vs reset vs revert

CommandEffect
git restoreUnstage or discard changes
git resetMove HEAD (destructive)
git revertSafe undo via new commit

Reset Types

git reset --soft HEAD~1   # keep changes staged
git reset --mixed HEAD~1  # default, unstage
git reset --hard HEAD~1   # discard everything

Reflog

git reflog
git reset --hard HEAD@{2}

Remote Repositories & Collaboration

Remote Commands

git remote add origin https://github.com/user/repo.git
git remote -v
git push -u origin main
git pull origin main

Pull vs Fetch

  • fetch: Download only
  • pull: Fetch + merge

Merge vs Rebase

Merge: Preserves history
Rebase: Linear history

git rebase main        # replay feature on main
git rebase -i HEAD~3   # interactive

40 Full Git & GitHub Interview Questions & Answers

Q1. What is Git and how is it different from SVN?

Git: Distributed, local commits, fast branching.
SVN: Centralized, server-based, slower.

Q2. What is the difference between git pull and git fetch?

fetch: Downloads commits, doesn't merge.
pull: Fetch + merge into current branch.

git fetch origin
git merge origin/main

Q3. What is the HEAD pointer?

Points to the current commit or branch tip. HEAD~1 = parent.

Q4. What are the three main areas of a Git project?

1. Working Directory
2. Staging Area (Index)
3. Repository (.git)

Q5. How do you undo the last commit?

Keep changes: git reset --soft HEAD~1
Discard: git reset --hard HEAD~1

Q6. Difference between git reset and git revert?

reset: Moves HEAD, rewrites history (unsafe for shared).
revert: Creates new commit undoing changes (safe).

git revert abc123

Q7. What is git stash?

Temporarily saves uncommitted changes. Use before switching branches.

git stash push -m "WIP"
git stash apply

Q8. How do you resolve merge conflicts?

Edit conflicted files → remove markers → git add → git commit.

Q9. What is rebase and when to avoid it?

Rewrites history by moving commits. Avoid on shared branches.

git rebase main  # don't do on pushed branch

Q10. What is .gitignore?

Specifies files Git should ignore.

node_modules/
.env
*.log

Q11. How to delete a local branch?

git branch -d feature/login

Q12. How to force push safely?

Only on your feature branch. Never on main.

git push --force-with-lease

Q13. What is git cherry-pick?

Apply a specific commit from another branch.

git cherry-pick abc123

Q14. What is interactive rebase?

Clean up history: squash, edit, reorder commits.

git rebase -i HEAD~3

Q15. How to view commit history?

git log --oneline --graph --all

Q16. What is a detached HEAD?

HEAD points to commit, not branch. Changes may be lost.

Q17. How to amend last commit?

git commit --amend -m "New message"

Q18. What is git blame?

Shows who changed each line.

git blame index.html

Q19. How to create a tag?

git tag -a v1.0.0 -m "Release"

Q20. What is git bisect?

Binary search to find bug-introducing commit.

Q21. How to squash commits?

git rebase -i HEAD~3 → change "pick" to "squash"

Q22. What is the difference between fork and clone?

Fork: Copy on GitHub.
Clone: Local copy.

Q23. How to keep fork in sync?

git remote add upstream original-repo
git fetch upstream
git rebase upstream/main

Q24. What is a Pull Request?

Request to merge changes. Enables code review.

Q25. How to resolve PR conflicts?

Pull main → merge locally → resolve → push.

Q26. What is git hook?

Scripts that run on events (pre-commit, post-merge).

Q27. How to ignore tracked files?

git rm --cached file.txt
echo "file.txt" >> .gitignore

Q28. What is git submodule?

Include another Git repo as subdirectory.

Q29. How to view remote branches?

git branch -r

Q30. What is git worktree?

Checkout multiple branches simultaneously.

Q31. How to clean untracked files?

git clean -fd

Q32. What is git archive?

Create zip/tar of specific commit.

Q33. How to sign commits?

git commit -S -m "Signed"

Q34. What is GitHub Actions?

CI/CD platform. Automate build, test, deploy.

Q35. How to protect main branch?

Branch protection rules: require PR, status checks.

Q36. What is git lfs?

Large File Storage for binaries (videos, models).

Q37. How to undo git add?

git restore --staged file.js

Q38. What is git shortlog?

Summary of commits by author.

Q39. How to create an empty branch?

git checkout --orphan new-branch

Q40. What is the best commit message format?

feat: add user authentication
fix: resolve null pointer in login
docs: update API reference

Git Best Practices

  • Commit often, small changes
  • Use feat:, fix: prefixes
  • Always pull before push
  • Never force push to main
  • Use feature branches
  • Write clear PR descriptions
  • Add .gitignore early
  • Review code before merging

Key Takeaways

  • Git = local + distributed
  • Use revert for public history
  • Rebase for clean history, merge for safety
  • Always configure .gitignore
  • Feature branches + PRs = best workflow
  • Never commit secrets
