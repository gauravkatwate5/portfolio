Structure, Tags, Elements, Attributes

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a web page using elements represented by tags.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title</title>
</head>
<body>
  <h1>Heading</h1>
  <p class="intro" id="first">Paragraph with attributes</p>
</body>
</html>

Key Points: <!DOCTYPE html> declares HTML5. lang attribute improves accessibility. Global attributes: id, class, style, title, data-*.

Semantic HTML

Semantic elements clearly describe their meaning to both the browser and the developer.

<header>
  <nav><a href="/">Home</a></nav>
</header>
<main>
  <section>
    <article>
      <h2>Blog Post</h2>
      <p>Content...</p>
    </article>
  </section>
  <aside>Sidebar</aside>
</main>
<footer>
  © 2025 Company
</footer>

Benefits: Better SEO, accessibility (screen readers), maintainable code.

Forms, Inputs, Buttons, Validation

<form action="/submit" method="POST" novalidate>
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required minlength="2" maxlength="50" placeholder="John Doe">

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="password">Password:</label>
  <input type="password" id="password" name="password" required pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}">

  <button type="submit">Submit</button>
</form>

Validation Attributes: required, minlength, maxlength, pattern, type="email", type="url", autocomplete.

Flexbox

.container {
  display: flex;
  flex-direction: row; /* row-reverse, column, column-reverse */
  justify-content: space-between; /* flex-start, center, space-around */
  align-items: center; /* stretch, flex-start, baseline */
  flex-wrap: wrap;
  gap: 1rem;
}

.item {
  flex: 1; /* flex-grow: 1; flex-shrink: 1; flex-basis: 0% */
  order: 2; /* Default 0 */
}

40+ Real Interview Questions with Full Explanations & Code

Q1. What is the difference between <div> and <section>?

<div> is a generic block container with no semantic meaning. <section> represents a thematic grouping of content, typically with a heading. Use <section> when content forms a logical section (e.g., chapter, tab).

<!-- Not semantic -->
<div><h2>About Us</h2><p>...</p></div>

<!-- Semantic -->
<section>
  <h2>About Us</h2>
  <p>We are a team...</p>
</section>

Q2. Explain the HTML5 doctype and its importance.

The <!DOCTYPE html> triggers standards mode in browsers, ensuring consistent rendering. Without it, browsers use quirks mode (legacy behavior).

<!DOCTYPE html> <!-- Must be first line -->
<html lang="en"><!-- Rest of document --></html>

Q3. What are data attributes? Give an example.

Custom attributes prefixed with data-* to store extra information. Accessible via JavaScript dataset.

<div id="user" data-id="123" data-role="admin">Gaurav</div>

<script>
  const user = document.getElementById('user');
  console.log(user.dataset.id); // "123"
</script>

Q4. How does label improve accessibility?

Clicking a <label> focuses its associated input. Screen readers announce the label text.

<label for="email">Email Address:</label>
<input type="email" id="email" name="email">

Q5. What is the purpose of alt attribute in <img>?

Provides alternative text if image fails to load. Essential for accessibility and SEO.

<img src="logo.png" alt="Company Logo - Blue Circle with Text">

Q6. Difference between <article> and <section>?

<article> is standalone content (blog post, news). <section> groups related content within a document.

<article>
  <h2>Blog Post</h2>
  <section><h3>Introduction</h3></section>
  <section><h3>Conclusion</h3></section>
</article>

Q7. What is the viewport meta tag?

Controls layout on mobile browsers. Without it, mobile zooms out.

<meta name="viewport" content="width=device-width, initial-scale=1">

Q8. Explain form validation attributes.

HTML5 provides built-in validation: required, pattern, min, max, type="email", etc.

<input type="text" required pattern="[A-Za-z]+" minlength="3">

Q9. What is the CSS box model?

Content â†’ Padding â†’ Border â†’ Margin. Default content-box. Use border-box for easier sizing.

* { box-sizing: border-box; }

Q10. Difference between margin and padding?

margin: outside element. padding: inside. Margin can collapse vertically.

Q11. What is position: static vs relative vs absolute?

static: default. relative: offset from itself. absolute: removed from flow, positioned relative to nearest positioned ancestor.

.relative { position: relative; top: 10px; }
.absolute { position: absolute; top: 0; right: 0; }

Q12. How does z-index work?

Controls stacking order of positioned elements. Higher values appear on top. Only works on position not static.

Q13. What is Flexbox? Explain justify-content.

1D layout model. justify-content aligns flex items along main axis.

.flex { display: flex; justify-content: space-between; }

Q14. When to use Flexbox vs Grid?

Flexbox: 1D ( (rows or columns). Grid: 2D (rows and columns). Use Grid for page layout, Flexbox for components.

Q15. Explain grid-template-areas.

Named grid areas for intuitive layout.

grid-template-areas: "header header" "nav main" "footer footer";

Q16. What are CSS custom properties (variables)?

Reusable values defined in :root or locally.

:root { --main-color: #4ec9b0; }
.btn { background: var(--main-color); }

Q17. Difference between transition and animation?

transition: state change (hover). animation: keyframe-based, runs automatically.

Q18. What is the CSS cascade?

Specificity + source order determines which rule applies. Inline > ID > Class > Element.

Q19. What does !important do? When to avoid?

Highest priority. Avoid â€” breaks cascade, hard to override.

Q20. Explain media queries for responsive design.

Apply CSS based on device characteristics.

@media (max-width: 768px) { .sidebar { display: none; } }

Q21. What is Bootstrapâ€™s grid system based on?

Q23. What is the difference between .container and .container-fluid?

.container: fixed width, changes at breakpoints. .container-fluid: full width always.

Q24. How to create a responsive navbar in Bootstrap 5?

Use navbar-expand-lg and data-bs-toggle="collapse".

Q25. What are utility classes in Bootstrap?

Single-purpose classes: mt-3, text-center, d-none, etc.

Q26. How to override Bootstrap styles?

Load custom CSS after Bootstrap. Use higher specificity or !important (last resort).

Q27. What is the role of data-bs-target in modals?

Links button to modal via ID. data-bs-toggle="modal" triggers it.

Q28. How to disable a button in Bootstrap?

<button class="btn btn-primary" disabled>Disabled</button>

Q29. What is shadow utility in Bootstrap?

Adds box shadows: shadow-sm, shadow, shadow-lg, shadow-none.

Q30. Explain Bootstrap 5 form validation.

Use needs-validation class and novalidate on form. Add is-valid/is-invalid via JS.

Q35. What is popover and how to initialize?

Tooltip-like content. Initialize with JS: new bootstrap.Popover(element).

Q36. How to use Bootstrap icons?

<i class="bi bi-heart"></i>

Q37. What is the visually-hidden class?

Hides element visually but keeps it accessible to screen readers.

Q38. How to create a sticky footer in Bootstrap?

Use flexbox on <body> with min-vh-100.

Q39. What are Sass variables in Bootstrap?

Bootstrap uses Sass. Customize via $primary: #yourcolor; before importing.

Q40. How to disable responsive features in Bootstrap?

Remove viewport meta tag and avoid using responsive classes (col-md-*).

Q41. (Bonus) What is the rel="noopener" in links?

Prevents security vulnerability when using target="_blank". Improves performance.

<a href="..." target="_blank" rel="noopener">Link</a>
