Basics

What is MongoDB?

MongoDB is a NoSQL, document-oriented database that stores data in flexible, JSON-like BSON format. It is schema-less, horizontally scalable, and designed for high performance and availability.

Advantages over SQL

MongoDB (NoSQL)SQL (Relational)
Schema-lessFixed schema
Horizontal scalingVertical scaling
JSON-like documentsTables with rows
Fast for nested dataJoins for relations
Great for unstructured dataACID transactions

Databases, Collections, Documents

  • Database: Container for collections
  • Collection: Group of documents (like a table)
  • Document: JSON-like object (like a row)
// Document example
{
  _id: ObjectId("507f1f77bcf86cd799439011"),
  name: "Gaurav",
  email: "gaurav@example.com",
  age: 25,
  hobbies: ["coding", "cricket"],
  address: { city: "Pune", state: "MH" }
}

MongoDB Data Types & BSON

BSON (Binary JSON) extends JSON with additional types:

  • String, Number, Boolean
  • ObjectId, Date, Array, Null
  • Binary, Regex, Timestamp

CRUD Operations (Native Driver)

insertOne

db.users.insertOne({
  name: "Alice",
  email: "alice@example.com",
  age: 30
})

find

// Find all
db.users.find()

// Find with condition
db.users.find({ age: { $gte: 25 } })

updateOne / updateMany

db.users.updateOne(
  { _id: ObjectId("...") },
  { $set: { age: 26 }, $push: { hobbies: "reading" } }
)

deleteOne

db.users.deleteOne({ email: "old@example.com" })

Query Operators

$eq, $ne, $gt, $gte, $lt, $lte
$in, $nin, $exists, $type
$regex, $or, $and, $not

Projection & Sorting

// Projection: select fields
db.users.find({}, { name: 1, email: 1, _id: 0 })

// Sorting
db.users.find().sort({ age: -1 })  // descending
db.users.find().limit(10).skip(20)

Mongoose ORM

Installation & Connection

npm install mongoose

// db.js
const mongoose = require('mongoose');
require('dotenv').config();

mongoose.connect(process.env.MONGO_URI, {
  useNewUrlParser: true,
  useUnifiedTopology: true
})
.then(() => console.log('MongoDB Connected'))
.catch(err => console.error(err));

Defining Schema & Model

const userSchema = new mongoose.Schema({
  name: { type: String, required: true, trim: true },
  email: { type: String, required: true, unique: true, lowercase: true },
  age: { type: Number, min: 0, max: 120 },
  hobbies: [String],
  isActive: { type: Boolean, default: true },
  createdAt: { type: Date, default: Date.now }
});

const User = mongoose.model('User', userSchema);

CRUD with Mongoose

// Create
const user = await User.create({ name: 'Bob', email: 'bob@example.com' });

// Read
const users = await User.find({ age: { $gt: 25 } }).select('name email');

// Update
await User.updateOne({ _id: id }, { $set: { age: 30 } });

// Delete
await User.deleteOne({ _id: id });

Schema Validation & Hooks

// Custom validator
email: {
  type: String,
  validate: {
    validator: v => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(v),
    message: 'Invalid email'
  }
}

// Pre-save hook
userSchema.pre('save', async function(next) {
  if (this.isModified('password')) {
    this.password = await bcrypt.hash(this.password, 10);
  }
  next();
});

Relationships & populate()

// Post Schema
const postSchema = new mongoose.Schema({
  title: String,
  content: String,
  author: { type: mongoose.Schema.Types.ObjectId, ref: 'User' }
});

const Post = mongoose.model('Post', postSchema);

// Populate
const posts = await Post.find().populate('author', 'name email');

Advanced Topics

Aggregation Pipeline

db.users.aggregate([
  { $match: { age: { $gte: 18 } } },
  { $group: { _id: "$city", total: { $sum: 1 }, avgAge: { $avg: "$age" } } },
  { $sort: { total: -1 } },
  { $limit: 5 }
])

Indexing

// Single field
db.users.createIndex({ email: 1 }, { unique: true })

// Compound
db.users.createIndex({ city: 1, age: -1 })

// Text index
db.articles.createIndex({ title: "text", content: "text" })

40 Full MongoDB + Mongoose Interview Questions & Answers

Q1. What is the difference between SQL and NoSQL databases?

SQL (Relational): Uses tables with fixed schema, supports ACID, uses JOINs, vertically scalable.
NoSQL (MongoDB): Uses collections of documents, schema-less, supports eventual consistency, horizontally scalable, no JOINs (uses embedding or referencing).

MongoDB is ideal for unstructured, rapidly changing data like logs, user profiles, real-time analytics.

Q2. What is BSON and why is it used?

BSON (Binary JSON) is a binary serialization format used to store documents in MongoDB. It extends JSON by adding support for types like Date, ObjectId, Binary, etc. BSON enables efficient traversal and storage compared to plain JSON.

{ "name": "John", "age": 30, "created": new Date() } → BSON binary

Q3. What is ObjectId in MongoDB?

ObjectId is a 12-byte unique identifier for documents. It contains:

  • 4 bytes: timestamp
  • 5 bytes: random value
  • 3 bytes: incrementing counter

It is globally unique and sortable by creation time.

ObjectId("507f1f77bcf86cd799439011")

Q4. What are indexes in MongoDB? Why are they important?

Indexes support efficient query execution. Without index, MongoDB scans every document (Collection Scan). With index, it uses Index Scan.

Types: single, compound, text, geospatial, TTL.

db.users.createIndex({ email: 1 }) // B-tree index
db.logs.createIndex({ createdAt: 1 }, { expireAfterSeconds: 86400 }) // TTL

Q5. What is the Aggregation Pipeline?

A framework for data processing with stages like $match, $group, $project, $sort, $lookup. Each stage transforms documents and passes to the next.

db.sales.aggregate([
  { $match: { status: "A" } },
  { $group: { _id: "$cust_id", total: { $sum: "$amount" } } },
  { $sort: { total: -1 } }
])

Q6. What is populate() in Mongoose?

populate() replaces ObjectId references with actual documents from another collection (like SQL JOIN).

const posts = await Post.find().populate('author', 'name email');

Without populate: author: "60d5f..."
With populate: author: { name: "Gaurav", email: "g@..." }

Q7. What is MongoDB Atlas?

MongoDB Atlas is the official cloud-hosted MongoDB service. It provides automated backups, scaling, monitoring, and security. You get a connection string with mongodb+srv:// protocol.

mongodb+srv://user:pass@cluster0.xxxxx.mongodb.net/dbname

Q8. How to connect Mongoose to MongoDB Atlas?

Use the Atlas connection string in mongoose.connect(). Enable IP whitelist and create database user.

mongoose.connect(process.env.MONGO_URI)

Q9. What are Mongoose middlewares (hooks)?

Functions that run before or after certain operations like save, validate, remove.

schema.pre('save', function(next) {
  this.updatedAt = Date.now();
  next();
});

Q10. What is the difference between find() and findOne()?

find() returns a cursor (array-like) of all matching documents.
findOne() returns the first matching document or null.

await User.find({ age: 25 })     // [doc1, doc2]
await User.findOne({ email: 'x' }) // doc or null

Q11. What is $lookup in aggregation?

Performs a left outer join with another collection.

{
  $lookup: {
    from: "users",
    localField: "userId",
    foreignField: "_id",
    as: "userDetails"
  }
}

Q12. How to embedded vs reference data modeling?

Embedded: Store related data in same document (denormalized). Fast reads.
Reference: Store ObjectId, use populate() or $lookup. Flexible, avoids duplication.

Q13. What is sharding in MongoDB?

Horizontal partitioning of data across multiple servers. Each shard holds a portion of data. Enables massive scale.

Q14. What is replica set?

Group of mongod instances that maintain the same data. Provides redundancy and high availability. One primary, multiple secondaries.

Q15. How to create a text index?

For full-text search.

db.articles.createIndex({ title: "text", content: "text" })
db.articles.find({ $text: { $search: "mongodb" } })

Q16. What is TTL index?

Automatically removes documents after a specified time.

db.sessions.createIndex({ createdAt: 1 }, { expireAfterSeconds: 3600 })

Q17. How to handle transactions in MongoDB?

Multi-document ACID transactions (since v4.0) in replica sets.

const session = await mongoose.startSession();
session.startTransaction();
try {
  await Account.updateOne({ _id: a }, { $inc: { balance: -100 } }, { session });
  await Account.updateOne({ _id: b }, { $inc: { balance: 100 } }, { session });
  await session.commitTransaction();
} catch (err) {
  await session.abortTransaction();
}

Q18. What is lean() in Mongoose?

Returns plain JavaScript objects instead of Mongoose documents. Faster, no methods like .save().

await User.find().lean()

Q19. How to validate data in Mongoose?

Using schema types, required, enum, min/max, custom validators.

Q20. What is discriminator in Mongoose?

Allows schema inheritance. Different models share the same collection.

const baseSchema = new Schema({ kind: String });
const User = mongoose.model('Person', baseSchema);
const Employee = User.discriminator('Employee', new Schema({ dept: String }));

Q21. How to paginate results?

Use .skip() and .limit().

const page = 2, size = 10;
await User.find().skip((page-1)*size).limit(size);

Q22. What is explain()?

Shows query execution plan: index used, documents scanned, etc.

db.users.find({ age: 30 }).explain("executionStats")

Q23. How to backup MongoDB?

Use mongodump for logical backup, Atlas automated backups, or EBS snapshots.

Q24. What is the N+1 query problem?

Fetching related data in a loop causes N+1 queries. Solve with populate or aggregation.

Q25. How to use geospatial queries?

Use 2dsphere index.

db.places.createIndex({ location: "2dsphere" })
db.places.find({
  location: {
    $near: {
      $geometry: { type: "Point", coordinates: [ -73.9, 40.7 ] },
      $maxDistance: 1000
    }
  }
})

Q26. What is change streams?

Real-time notification of database changes. Like triggers.

const changeStream = collection.watch();
changeStream.on('change', console.log);

Q27. How to handle large datasets?

Use cursors, aggregation with $out, $merge, or map-reduce.

Q28. What is schema versioning?

Add version field or use migration tools like migrate-mongo.

Q29. How to prevent NoSQL injection?

Never concatenate user input into queries. Use Mongoose (sanitizes) or validate input.

Q30. What is write concern?

Defines acknowledgment level for write operations (e.g., w: "majority").

Q31. What is read preference?

Controls which replica set member to read from: primary, secondary, etc.

Q32. How to optimize MongoDB performance?

• Proper indexing
• Use projection
• Avoid large documents (>16MB)
• Use covered queries
• Monitor with Atlas/Compass

Q33. What is $project in aggregation?

Reshapes documents: include, exclude, or compute fields.

{ $project: { name: 1, email: 1, _id: 0 } }

Q34. What is $unwind?

Deconstructs an array field into multiple documents.

{ $unwind: "$hobbies" }

Q35. How to create a unique compound index?

db.users.createIndex({ email: 1, phone: 1 }, { unique:":
true })

Q36. What is the difference between remove() and deleteOne()?

remove() is deprecated. deleteOne() deletes first match. deleteMany() deletes all.

Q37. How to use virtuals in Mongoose?

Computed fields not stored in DB.

schema.virtual('fullName').get(function() {
  return this.firstName + ' ' + this.lastName;
});

Q38. What is statics vs methods in Mongoose?

statics: Model-level functions.
methods: Instance-level functions.

schema.statics.findByEmail = function(email) { ... }
schema.methods.getInitials = function() { ... }

Q39. How to use MongoDB with TypeScript?

Define interfaces and use Mongoose with @types/mongoose.

Q40. How to deploy MongoDB in production?

Use MongoDB Atlas (recommended), or self-managed replica set with authentication, TLS, backups, and monitoring.

Key Takeaways

  • Always index frequently queried fields
  • Use populate() wisely to avoid N+1
  • Prefer embedding for 1:1, 1:few; reference for 1:many
  • Use aggregation for complex reporting
  • Enable authentication and TLS
  • Monitor slow queries with explain()
  • Backup regularly
  • Use Mongoose for validation and structure
