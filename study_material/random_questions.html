JavaScript Core Concepts

Q1. What is typeof null?

Answer: typeof null returns "object". This is a well-known historical bug in JavaScript introduced in the first version and preserved for backward compatibility. The original intent was to represent null as a separate type, but due to implementation, it was grouped under objects.

console.log(typeof null); // "object"

Best Practice: Always use strict equality (===) when checking for null:

if (value === null) { /* handle null */ }

Q2. What is the difference between undefined and not defined?

undefined: A variable has been declared but not assigned a value. It is a primitive value in JavaScript.

not defined: The variable has never been declared in the current scope, leading to a ReferenceError when accessed.

let x;           // declared, not initialized
console.log(x);  // undefined

console.log(y);  // ReferenceError: y is not defined

Real-world: Use typeof to safely check without throwing error:

if (typeof y === 'undefined') { /* safe */ }

Q3. What is NaN? How to check for it properly?

NaN stands for Not a Number. It is a special value representing an invalid number result (e.g., 0/0, "abc" * 5).

Crucially, NaN === NaN is false — it is not equal to itself.

console.log(0 / 0);           // NaN
console.log("hello" * 10);    // NaN
console.log(NaN === NaN);     // false

Correct way to check:

Number.isNaN(value)   // ES6, recommended
// or
isNaN(value)          // older, but coerces types

Q4. What is the difference between null and undefined?

null: An intentional absence of any object value. It is a primitive value assigned by the developer.

undefined: Represents an uninitialized or missing value. Occurs when a variable is declared but not assigned, or a function returns nothing.

let a = null;           // explicit empty value
let b;                  // automatically undefined
console.log(a === null);     // true
console.log(b === undefined); // true
console.log(a == b);         // true (loose)
console.log(a === b);        // false (strict)

Rule of Thumb: Use null for "no value", undefined for "not set".

Q5. What is destructuring in JavaScript?

Destructuring is a syntax that allows you to unpack values from arrays or properties from objects into distinct variables in a concise way.

// Array destructuring
const [first, second] = [10, 20];
console.log(first);  // 10

// Object destructuring
const { name, age } = { name: "Gaurav", age: 25 };
console.log(name);   // "Gaurav"

// With defaults and rest
const { city = "Pune", ...rest } = user;

Use Case: Clean API responses, function parameters.

Q6. Can you destructure nested objects?

Yes. You can nest destructuring patterns to any depth.

const user = {
  profile: {
    name: "Gaurav",
    address: { city: "Pune", pin: "411001" }
  }
};

const { profile: { name, address: { city } } } = user;
console.log(name, city); // Gaurav, Pune

With alias and default:

const { profile: { address: { pin: zip = '000000' } } } = user;

Q7. Is NaN equal to itself?

No. NaN is the only value in JavaScript not equal to itself.

console.log(NaN === NaN); // false

This is why Number.isNaN() exists.

Q8. Can JSON store functions?

No. JSON is a data format. It supports only:

  • strings
  • numbers
  • arrays
  • objects
  • booleans
  • null

Functions, undefined, and Symbol are lost on JSON.stringify().

const obj = { fn: () => {}, x: undefined };
JSON.stringify(obj); // "{}"

Q11. What are Error Boundaries in React?

Error Boundaries are React components that catch JavaScript errors anywhere in their child component tree, log them, and display a fallback UI instead of crashing the app.

Only work with class components using static getDerivedStateFromError() or componentDidCatch().

class ErrorBoundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, info) {
    logErrorToService(error, info);
  }

  render() {
    return this.state.hasError 
      ? 
Something went wrong.
 
      : this.props.children;
  }
}

Q12. Can hooks replace error boundaries?

No. Hooks cannot catch errors thrown in:

  • Render phase
  • Event handlers
  • Async code
  • Child component lifecycle

Error boundaries are still required for robust error handling.

Q14. What is a REST API?

REST (Representational State Transfer) is an architectural style for designing networked applications. It uses:

  • HTTP methods (GET, POST, PUT, DELETE)
  • Stateless communication
  • Resource-based URLs
  • JSON/XML payloads
GET    /api/users        → List users
POST   /api/users        → Create user
PUT    /api/users/1      → Update user
DELETE /api/users/1      → Delete user

Q15. What are common HTTP status codes?

2xx Success: 200 OK, 201 Created, 204 No Content
4xx Client Error: 400 Bad Request, 401 Unauthorized, 404 Not Found
5xx Server Error: 500 Internal Server Error, 503 Service Unavailable

Q16. Difference between 204 and 200?

200 OK: Request succeeded and response has a body.
204 No Content: Request succeeded but no response body (common in DELETE, or when no data to return).

// DELETE /users/1
res.status(204).send(); // No body

Q19. Describe require vs import

require (CommonJS):

  • Used in Node.js by default
  • Dynamic, synchronous
  • Can be called anywhere

import (ES6 Modules):

  • Static, hoisted
  • Supports tree-shaking
  • Requires "type": "module" in package.json
// CommonJS
const fs = require('fs');

// ES6
import fs from 'fs';

React & Frontend

Q20. What is React Router? How does it work?

React Router is the standard library for routing in React apps. It enables navigation between views without full page reloads using client-side routing.

import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

Uses HTML5 History API or hash routing.

Node.js & Backend

Q21. What is Node.js? Why use it for backend?

Node.js is a runtime environment that executes JavaScript outside the browser using Chrome’s V8 engine and libuv for async I/O.

Why for backend?

  • Non-blocking, event-driven I/O
  • Great for real-time apps (chat, streaming)
  • Full-stack JavaScript
  • Huge npm ecosystem
  • High performance for I/O-heavy tasks

Q22. Difference between synchronous and asynchronous programming?

Synchronous: Operations block execution until complete.
Asynchronous: Operations run in background; callbacks/promises handle results later.

// Sync - blocks
const data = fs.readFileSync('file.txt');
console.log(data);

// Async - non-blocking
fs.readFile('file.txt', (err, data) => {
  console.log(data);
});
console.log('This runs first');

Q23. What is the Event Loop in Node.js?

The event loop is the heart of Node.js async model. It continuously checks the call stack and task queues.

Call Stack (empty?) ↓ Microtask Queue (Promise.then, queueMicrotask) ↓ Macrotask Queue (setTimeout, setInterval, I/O) ↓ Repeat

Order: Call stack → Microtasks → One macrotask → Repeat.

Q24. What is the microtask queue?

Higher-priority queue than macrotasks. Contains:

  • Promise resolutions
  • queueMicrotask()
  • MutationObserver

Drained completely before macrotasks.

Q25. Can setTimeout be a microtask?

No. setTimeout is a macrotask. Microtasks run before the next macrotask.

console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Output: 1 4 3 2

Q26. Why does Promise run before setTimeout?

Microtasks (Promises) have higher priority than macrotasks (setTimeout) in the event loop.

After current call stack, all microtasks are processed before any macrotask.

Q27. Name three ways to handle async in Node.js

  1. Callbacks: fs.readFile(path, callback)
  2. Promises: fs.promises.readFile()
  3. async/await: const data = await readFile()

Q28. Difference between Promises and async/await?

Promises: Object-based async pattern.
async/await: Syntactic sugar over Promises for cleaner, synchronous-looking code.

// Promise
getUser(id)
  .then(user => getPosts(user.id))
  .then(posts => console.log(posts));

// async/await
const user = await getUser(id);
const posts = await getPosts(user.id);
console.log(posts);

Q29. What is middleware in Node.js?

Functions that have access to req, res, and next(). Used in Express for:

  • Logging
  • Authentication
  • Parsing JSON
  • Error handling
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
});

Q31. What is Mongoose? Why use it with MongoDB?

Mongoose is an ODM (Object Data Modeling) library for MongoDB and Node.js. It provides:

  • Schema definition and validation
  • Middleware (pre/post hooks)
  • Population (joins)
  • Query building
const userSchema = new Schema({
  name: { type: String, required: true },
  email: { type: String, unique: true }
});
const User = mongoose.model('User', userSchema);

Q32. Why use Mongoose instead of raw MongoDB driver?

Raw driver is low-level. Mongoose adds:

  • Type casting
  • Validation
  • Default values
  • Middleware
  • Population

Use raw driver for performance-critical apps.

Q34. Is Node.js single-threaded?

Yes, for JavaScript execution (event loop). But:

  • Uses libuv thread pool (default 4 threads) for I/O
  • CPU-heavy tasks can block

Use worker_threads or clustering for CPU work.

Q35. How does Node.js handle concurrency despite being single-threaded?

Through non-blocking I/O and the event loop. While waiting for DB/file/network, it processes other requests.

Thread pool handles blocking system calls (DNS, file I/O).

Q36. What is the control flow in Node.js?

  1. Execute current call stack
  2. Process all microtasks
  3. Process one macrotask
  4. Repeat

Q37. Main disadvantages of Node.js?

  • Single-threaded → CPU-heavy tasks block event loop
  • Callback hell (if not using Promises)
  • Immaturity in some enterprise tools
  • Not ideal for heavy computation

Q42. Why use Express.js?

Express is a minimal, unopinionated web framework that simplifies:

  • Routing
  • Middleware
  • JSON parsing
  • Error handling
  • Templating

Reduces boilerplate from raw http module.

Q43. What is event-driven programming in Node.js?

Code responds to events (request, file read, timer) via listeners.

const EventEmitter = require('events');
const emitter = new EventEmitter();

emitter.on('greet', (name) => console.log(`Hello ${name}`));
emitter.emit('greet', 'Gaurav');

Q44. What is Buffer in Node.js?

Buffer is a way to handle binary data (streams, files, sockets). It’s like an array of bytes.

const buf = Buffer.from('Hello');
console.log(buf);        // 
console.log(buf.toString()); // "Hello"

Q45. What are Streams in Node.js?

Streams process data in chunks. Types:

  • Readable: Input (file, HTTP request)
  • Writable: Output
  • Duplex: Both
  • Transform: Modify data

Efficient for large files.

Q46. What is callback hell?

Nested callbacks leading to unreadable, hard-to-maintain code.

fs.readFile('a.txt', (err, data) => {
  fs.writeFile('b.txt', data, () => {
    fs.readFile('b.txt', (err, data) => {
      // deeply nested
    });
  });
});

Solutions: Promises, async/await, named functions.

Q47. What is body-parser in Node.js?

Middleware to parse incoming request bodies (JSON, URL-encoded). Now built into Express:

app.use(express.json());
app.use(express.urlencoded({ extended: true }));

Q48. What is CORS? Why use it?

CORS (Cross-Origin Resource Sharing) is a security feature that allows or restricts cross-domain requests.

const cors = require('cors');
app.use(cors({ origin: 'https://myfrontend.com' }));

Prevents unauthorized domains from accessing your API.

Q53. Query params vs path params?

Path params: Part of URL path (required).

GET /users/123 → req.params.id = "123"

Query params: Optional filters after ?.

GET /users?active=true&page=2 → req.query

Use passport-google-oauth20 strategy with OAuth 2.0.

passport.use(new GoogleStrategy({
  clientID: process.env.GOOGLE_ID,
  clientSecret: process.env.GOOGLE_SECRET,
  callbackURL: "/auth/google/callback"

  // find or create user

Q55. What is WebSocket? How to use in Node.js?

WebSocket enables full-duplex, real-time communication over a single TCP connection.

Use ws or Socket.IO:

const WebSocket = require('ws');
const wss = new WebSocket.Server({ port: 8080 });

wss.on('connection', ws => {
  ws.on('message', msg => {
    wss.clients.forEach(client => client.send(msg));
  });
});

Q56. How to handle sessions in Node.js?

Use express-session with a store (Redis, MongoDB):

app.use(session({
  secret: 'my-secret',
  resave: false,
  saveUninitialized: false,
  store: MongoStore.create({ mongoUrl: '...' })
}));

Q57. What is Cluster in Node.js?

Allows using all CPU cores by forking worker processes.

const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
} else {
  // worker: start server
}

Q58. Regular expressions in MongoDB?

// Case-insensitive search for "gaurav"
db.users.find({ name: { $regex: /gaurav/i } })

// Starts with "Gau"
db.users.find({ name: { $regex: /^Gau/ } })

Q59. How to write a Student schema in Mongoose?

const studentSchema = new mongoose.Schema({
  name: {
    type: String,
    required: true,
    trim: true
  },
  email: {
    type: String,
    required: true,
    unique: true,
    lowercase: true
  },
  age: {
    type: Number,
    min: [5, 'Too young'],
    max: [100, 'Too old']
  },
  grade: {
    type: String,
    enum: ['A', 'B', 'C', 'D', 'F'],
    default: 'A'
  },
  enrolledAt: {
    type: Date,
    default: Date.now
  },
  isActive: {
    type: Boolean,
    default: true
  }
}, { timestamps: true });

const Student = mongoose.model('Student', studentSchema);

Key Takeaways

  • typeof null === "object" — know the bug
  • Use Number.isNaN(), not isNaN()
  • Always use === for strict comparison
  • Prefer async/await over nested callbacks
  • Understand event loop: microtasks before macrotasks
  • Use bcrypt for passwords, never plain text
  • Mongoose adds structure and safety to MongoDB
  • Express + middleware = clean, scalable APIs
  • CORS and CSRF are security essentials
  • Cluster for multi-core, streams for large data
