What is typeof null?

typeof null returns "object" due to a historical bug in JavaScript. Originally, values were stored with type tags, and null used the object tag (000). This was never fixed for backward compatibility.

console.log(typeof null); // "object"

Q: Is typeof null a bug?

Yes — a legacy bug from the first version of JavaScript. Should return "null", but returns "object" for compatibility.

undefined vs Not Defined

undefined: Variable declared but not assigned.
Not defined: Variable never declared → ReferenceError.

let a;
console.log(a); // undefined

console.log(b); // ReferenceError: b is not defined

Q: How to check if a variable is undefined?

if (typeof x === 'undefined') — safe even if x is not declared.

What is NaN? How to Check?

NaN = Not-a-Number. Result of invalid math (e.g., 0/0).

isNaN() coerces value. Number.isNaN() does not.

console.log(0 / 0); // NaN
console.log(isNaN("abc"));      // true (coerces)
console.log(Number.isNaN("abc")); // false
console.log(Number.isNaN(NaN));   // true

Q: Is NaN equal to itself?

No. NaN === NaN is false. Use Number.isNaN().

Q: Should you use null or undefined?

Use null for intentional absence. Let JS handle undefined naturally.

Destructuring

Extract values from arrays/objects into variables.

const user = { name: "Gaurav", city: "Pune" };
const { name, city } = user;
console.log(name, city); // Gaurav Pune

const arr = [1, 2, 3];
const [a, b] = arr;
console.log(a, b); // 1 2

Q: Can you destructure nested objects?

Yes: const { address: { street } } = user;

Why Use React?

  • Declarative UI
  • Component-based
  • Virtual DOM → fast updates
  • Rich ecosystem (React Router, Redux, Hooks)
function App() {
  return <h1>Hello React</h1>;
}

Q: React is a library or framework?

Library. You control the app structure.

What is JSX?

Syntax extension to write HTML-like code in JavaScript. Transpiled to React.createElement().

const element = <h2>Welcome, Gaurav!</h2>;
// Becomes:
React.createElement("h2", null, "Welcome, Gaurav!")

Q: Can you use JSX without React?

Technically yes, but it’s designed for React.

Error Boundaries in React

Catch JavaScript errors in child components. Only work in class components.

class ErrorBoundary extends React.Component {
  state = { hasError: false };
  static getDerivedStateFromError() {
    return { hasError: true };
  }
  render() {
    return this.state.hasError 
      ? <h3>Something went wrong!</h3> 
      : this.props.children;
  }
}

Q: Can hooks replace error boundaries?

No. Hooks cannot catch errors in render phase.

dependencies: runtime. devDependencies: build/test tools.

REST API

Architectural style using HTTP methods to perform CRUD.

app.get('/users', getUsers);
app.post('/users', createUser);
app.put('/users/:id', updateUser);
app.delete('/users/:id', deleteUser);

Q: What makes an API RESTful?

Stateless, cacheable, uniform interface, resource-based URLs.

Node.js for Backend

JS runtime on V8. Non-blocking I/O → great for APIs, real-time.

import express from "express";
const app = express();
app.get("/", (req, res) => res.send("Hello Node"));
app.listen(3000);

Q: Is Node.js single-threaded?

Yes (event loop), but uses libuv for async I/O.

Synchronous vs Asynchronous

Sync: blocks execution.
Async: continues while waiting.

console.log("Start");
setTimeout(() => console.log("Async"), 1000);
console.log("End");
// Output: Start → End → Async

Q: Name three ways to handle async in Node.js

Callbacks, Promises, async/await.

Event Loop in Node.js

Handles async callbacks. Phases: timers → I/O → poll → check → microtasks.

console.log("A");
setTimeout(() => console.log("B"), 0);
Promise.resolve().then(() => console.log("C"));
console.log("D");
// Output: A D C B

Q: Why does Promise run before setTimeout?

Microtask queue has higher priority than callback queue.

Microtask Queue

Higher priority than macrotask (callback) queue. Used by Promises, queueMicrotask.

Promise.resolve().then(() => console.log("micro"));
setTimeout(() => console.log("macro"), 0);
// micro → macro

Q: Can setTimeout be a microtask?

No. Only Promises and MutationObserver.

React Router

Client-side routing for SPAs.

import { BrowserRouter, Routes, Route } from "react-router-dom";

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

Q: How to get URL params?

useParams() hook: const { id } = useParams();

Middleware in Node.js

Functions with access to req, res, next.

const logger = (req, res, next) => {
  console.log(\`\${req.method} \${req.url}\`);
  next();
};
app.use(logger);

Q: What happens if you forget next()?

Request hangs — response never sent.

JSON (JavaScript Object Notation)

Lightweight data format. Used in APIs.

const obj = { name: "Gaurav", city: "Pune" };
const json = JSON.stringify(obj);
console.log(json); // {"name":"Gaurav","city":"Pune"}
console.log(JSON.parse(json)); // object

Q: Can JSON store functions?

No. Only data: strings, numbers, arrays, objects, boolean, null.

Key Takeaways

  • typeof null === "object" → historical bug
  • undefined ≠ not defined
  • NaN === NaN is false → use Number.isNaN()
  • null = intentional, undefined = uninitialized
  • React = declarative, component-based, Virtual DOM
  • JSX → React.createElement()
  • Node.js = non-blocking, event-driven
  • Event Loop: microtasks > macrotasks
  • Always call next() in middleware
  • REST: stateless, resource-based, HTTP verbs
