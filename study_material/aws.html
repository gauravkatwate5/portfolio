40 Full AWS Interview Questions & Answers

Q1. What is the difference between EC2 and Lambda?

EC2: Virtual servers you manage. Full control over OS, runtime, scaling. Pay per hour/second.
Lambda: Serverless functions. No server management. Auto-scales. Pay per invocation (ms). Max 15 mins.

Use EC2 for long-running apps, SSH, custom binaries.
Use Lambda for event-driven tasks, microservices, cron jobs.

# EC2: SSH into instance
ssh -i key.pem ec2-user@ec2-x-x-x-x.compute-1.amazonaws.com

# Lambda: Triggered by S3 upload
exports.handler = async (event) => { ... };

Q2. How do you make an S3 bucket public?

1. Disable "Block all public access"
2. Add bucket policy allowing s3:GetObject

{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::my-static-site/*"
  }]
}

Q3. How does auto-scaling work with load balancing?

ASG monitors CloudWatch metrics (CPU > 70%). When threshold is hit, it launches new EC2 instances from launch template. ELB automatically registers new instances via health checks. Traffic is distributed.

ASG → CloudWatch Alarm → Scale Out → New EC2 → ELB Health Check → In Service

Q5. What is IAM Role vs IAM Policy?

Policy: JSON document defining permissions.
Role: Identity that can be assumed by AWS services (EC2, Lambda). You attach policies to roles.

# Role for Lambda
Trust Policy: Allow lambda.amazonaws.com to assume
Attached Policy: Allow s3:PutObject

Q6. How to deploy a Node.js/React app on AWS?

Backend: Elastic Beanstalk or ECS
Frontend: S3 + CloudFront
DB: RDS or DynamoDB
Domain: Route 53
SSL: ACM
CI/CD: CodePipeline

Q7. Explain AWS Regions and AZs.

Region: Geographic location (e.g., ap-south-1 Mumbai)
AZ: Isolated data center within region (e.g., ap-south-1a)
For HA: Deploy RDS Multi-AZ, EC2 in 2+ AZs behind ELB.

Q8. How can you reduce AWS costs in production?

  • Use Spot Instances for non-critical workloads
  • Right-size EC2 (use CloudWatch)
  • Delete unused EBS volumes/snapshots
  • S3 lifecycle → Glacier after 30 days
  • Use Reserved Instances for predictable loads
  • Enable S3 Intelligent-Tiering

Q10. How does CloudFront improve performance?

Caches content at 200+ edge locations. Users get data from nearest PoP, reducing latency. Supports invalidations for updates.

Origin: S3 bucket
Cache Behavior: /api/* → forward to API Gateway
SSL: ACM certificate

Q11. What is the purpose of Route 53 Alias records?

Route traffic to AWS resources (S3, ELB) without public IP. Free, supports health checks.

Alias: app.example.com → my-elb-123.elb.amazonaws.com

Q12. How to enable HTTPS for a static site on S3 + CloudFront?

1. Request ACM certificate
2. Configure CloudFront with ACM cert
3. Set Viewer Protocol Policy: Redirect HTTP to HTTPS

Q13. What is VPC Peering?

Private connection between two VPCs. Non-transitive. Uses AWS backbone, not internet.

Q14. When to use NAT Gateway vs NAT Instance?

NAT Gateway: Managed, highly available, auto-scaling.
NAT Instance: Cheaper, more control (e.g., SSH jump host).

Q15. How to back up an EC2 instance?

Create AMI (image) or snapshot EBS volumes. Automate with Lifecycle Manager.

aws ec2 create-image --instance-id i-1234567890abcdef0 --name "Backup"

Q16. What is the difference between Stop and Terminate in EC2?

Stop: Instance shuts down, EBS preserved, can restart.
Terminate: Instance deleted, root EBS deleted (unless configured).

Q17. How to monitor API Gateway latency?

Use CloudWatch metrics: Latency, 4XXError, 5XXError. Set alarms.

Q18. What is the AWS Well-Architected Framework?

Best practices in 6 pillars: Operational Excellence, Security, Reliability, Performance, Cost, Sustainability.

Q19. How to rotate IAM access keys safely?

1. Create new key
2. Update applications
3. Deactivate old key
4. Delete old key

Q20. What is S3 Transfer Acceleration?

Uses CloudFront edge locations to speed up uploads to S3 over long distances.

Q21. How to encrypt data at rest in RDS?

Enable encryption during creation using AWS KMS. Cannot enable later.

Q22. What is the difference between EBS and Instance Store?

EBS: Persistent block storage, survives reboot.
Instance Store: Ephemeral, high IOPS, lost on stop/terminate.

Q23. How to implement zero-downtime deployment?

Use Blue/Green deployment with Elastic Beanstalk or CodeDeploy. Route 53 weighted routing.

Q24. What is AWS Shield?

DDoS protection. Standard (free) for all, Advanced (paid) for WAF + 24/7 support.

Q25. How to share an AMI across accounts?

Modify AMI attributes to share with specific AWS account IDs.

Q26. What is the purpose of Placement Groups?

Control instance placement: Cluster (low latency), Spread (fault tolerance), Partition.

Q27. How to debug a failing Lambda function?

Check CloudWatch Logs, X-Ray traces, test locally with SAM CLI.

Q28. What is AWS Systems Manager Parameter Store?

Secure storage for configuration data and secrets. Free tier available.

Q29. How to automate backups of DynamoDB?

Enable Point-in-Time Recovery (PITR) or use on-demand backups.

Q30. What is the difference between horizontal and vertical scaling?

Vertical: Bigger instance (e.g., t3.micro → m5.large)
Horizontal: More instances (ASG)

Q31. How to restrict S3 bucket access to a specific VPC?

Use VPC Endpoint for S3 + bucket policy with aws:SourceVpce condition.

Q32. What is AWS Certificate Manager (ACM)?

Free SSL/TLS certificates for AWS services (ELB, CloudFront, API Gateway).

Q33. How to trigger Lambda from S3 upload?

Add S3 event notification → Lambda function ARN.

Q34. What is the AWS Free Tier?

12 months: 750 hrs t2.micro, 5GB S3, etc. Always-free: Lambda 1M requests, DynamoDB 25GB.

Q35. How to use AWS CLI with MFA?

Use aws sts get-session-token --serial-number arn:... --token-code 123456

Q36. What is Amazon Macie?

ML-powered service to discover, classify, and protect sensitive data in S3.

Q37. How to enable Cross-Region Replication in S3?

Enable versioning on both buckets → Create replication rule.

Q38. What is AWS Snowball?

Physical device to transfer petabytes of data into/out of AWS.

Q39. How to run Docker containers on AWS?

Use ECS (orchestration) or EKS (Kubernetes). Fargate for serverless containers.

Key Takeaways

  • Use IAM roles, never long-term keys
  • Design for Multi-AZ and auto-scaling
  • S3 + CloudFront for static sites
  • Automate everything with IaC
  • Monitor with CloudWatch, audit with CloudTrail
  • Secure with least privilege and encryption
  • Optimize costs with right-sizing and lifecycle policies
