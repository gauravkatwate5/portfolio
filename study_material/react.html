React Basics

What is React?

React is a JavaScript library for building user interfaces using components. It uses a declarative approach and efficient DOM updates via the Virtual DOM.

JSX

JavaScript XML — syntax extension that lets you write HTML-like code in JavaScript. Transpiled by Babel.

const element = <h1 className="title">Hello React!</h1>;
return <div>{element}</div>;

Virtual DOM

A lightweight in-memory representation of the real DOM. React computes the difference (diffing) and updates only changed parts.

Components

Reusable, self-contained pieces of UI. Can be functional or class-based.

Functional vs Class Components

FeatureFunctionalClass
SyntaxArrow/regular functionES6 class
StateHooks (useState)this.state
LifecycleuseEffectcomponentDidMount etc.
PerformanceLighterHeavier

Props

Read-only data passed from parent to child. Immutable.

// Parent
<UserCard name="Gaurav" role="Developer" />

// Child
function UserCard({ name, role }) {
  return <div>{name} - {role}</div>;
}

State

Mutable data managed within a component. Triggers re-render on change.

const [count, setCount] = useState(0);
<button onClick={() => setCount(count + 1)}>{count}</button>

React Hooks

useState

Adds state to functional components.

const [name, setName] = useState('Gaurav');
setName('Raj');

useEffect

Handles side effects (API calls, subscriptions). Runs after render.

useEffect(() => {
  // Runs on mount + dependency change
  fetchData();
  return () => {
    // Cleanup (unmount)
    cancelRequest();
  };
}, [id]); // Dependency array

useRef

Access DOM elements or persist values across renders without causing re-renders.

const inputRef = useRef(null);
<input ref={inputRef} />
inputRef.current.focus();

useContext

Consume context values without prop drilling.

const theme = useContext(ThemeContext);
return <div className={theme}>Content</div>;

useReducer

For complex state logic. Alternative to useState.

function reducer(state, action) {
  switch (action.type) {
    case 'increment': return { count: state.count + 1 };
  }
}
const [state, dispatch] = useReducer(reducer, { count: 0 });

useMemo

Memoizes expensive calculations. Recalculates only when dependencies change.

const expensiveValue = useMemo(() => computeExpensive(id), [id]);

useCallback

Memoizes functions to prevent unnecessary re-renders in child components.

const handleClick = useCallback(() => {
  saveData();
}, [data]);

Dependency Arrays & Cleanup

  • []: Run once on mount
  • [dep]: Run on mount + when dep changes
  • No array: Run on every render
  • Return function from useEffect → cleanup on unmount or before next run

Advanced Concepts

Higher Order Components (HOC)

Function that takes a component and returns an enhanced version.

function withAuth(WrappedComponent) {
  return function EnhancedComponent(props) {
    const { user } = useAuth();
    if (!user) return <Login />;
    return <WrappedComponent {...props} user={user} />;
  };
}

const ProtectedPage = withAuth(Dashboard);

Error Boundaries

Catch JavaScript errors in child component tree.

class ErrorBoundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong.</h1>;
    }
    return this.props.children;
  }
}

Lazy Loading & Suspense

const LazyComponent = React.lazy(() => import('./HeavyComponent'));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <LazyComponent />
    </Suspense>
  );
}

Performance Optimization

  • React.memo: Prevent re-render if props unchanged
  • useMemo: Cache expensive computations
  • useCallback: Cache functions
  • Code splitting with React.lazy
  • Avoid inline functions/objects in render
const MemoizedChild = React.memo(function Child({ onClick }) {
  return <button onClick={onClick}>Click</button>;
});

40+ React Interview Questions & Answers

Q1. What is the Virtual DOM?

A lightweight copy of the real DOM kept in memory. React updates the Virtual DOM first, computes the difference (diffing), and applies only necessary changes to the real DOM.

// React reconciles changes efficiently
setState → Virtual DOM update → Diff → Real DOM patch

Q2. Why is key prop important in lists?

Helps React identify which items changed, added, or removed. Without stable keys, React may re-render entire list inefficiently.

{users.map(user => (
  <User key={user.id} name={user.name} />
))}

Q3. What happens when you call setState?

Triggers a re-render. React batches multiple setState calls for performance. In event handlers, updates are async.

setCount(count + 1); // count is stale
setCount(prev => prev + 1); // functional update

Q4. Explain useEffect dependency array.

Controls when effect runs:

  • []: Run once on mount
  • [a, b]: Run on mount + when a or b changes
  • No array: Run after every render
useEffect(() => {
  fetchData();
}, [id]);

Q5. When does cleanup function in useEffect run?

Before the component unmounts and before the effect re-runs (if dependencies change).

useEffect(() => {
  const timer = setInterval(() => {}, 1000);
  return () => clearInterval(timer);
}, []);

Q6. What is useRef used for?

Two main uses:

  1. Access DOM elements: inputRef.current.focus()
  2. Persist mutable values across renders without causing re-renders
const countRef = useRef(0);
countRef.current++; // No re-render

Q7. Difference between useLayoutEffect and useEffect?

useLayoutEffect: Runs synchronously after DOM mutations, before paint. Use for DOM measurements. useEffect: Runs after paint.

Q8. How to optimize performance in React?

• Use React.memo
• useMemo and useCallback
• Code splitting with React.lazy
• Avoid inline functions in render
• Use production build

Q9. What is React.memo?

Shallow-compares props and skips re-render if they haven't changed.

const MyComponent = React.memo(function MyComponent(props) {
  /* render using props */
});

Q10. When to use useReducer vs useState?

Use useReducer when:

  • State logic is complex
  • Next state depends on previous
  • Multiple sub-values
  • Actions are dispatched

Q11. What are custom hooks?

Reusable functions that encapsulate logic using built-in hooks. Name must start with use.

function useFetch(url) {
  const [data, setData] = useState(null);
  useEffect(() => {
    fetch(url).then(res => res.json()).then(setData);
  }, [url]);
  return data;
}

Q12. How does React Router v6 differ from v5?

v6 uses <Routes> and element prop instead of component. No more Switch, exact. Better nesting.

Q13. What is Suspense?

Shows fallback UI while waiting for something (lazy components, data fetching with React 18+).

<Suspense fallback={<Spinner />}>
  <LazyDashboard />
</Suspense>

Q14. What are Error Boundaries?

Class components that catch errors in child tree using static getDerivedStateFromError and componentDidCatch.

Q15. How to prevent memory leaks in React?

Cleanup in useEffect: cancel API calls, clear intervals/timers, unsubscribe from events.

Q16. What is prop drilling? How to avoid?

Passing props through many layers. Avoid with Context API or state management.

Q17. What is the difference between state and props?

StateProps
MutableImmutable
Owned by componentPassed from parent
Triggers re-renderReceived on render

Q18. Explain React reconciliation.

Process of comparing old and new Virtual DOM trees and updating real DOM efficiently using heuristics (same type, key).

Q19. What is useImperativeHandle?

Customizes the instance value exposed when using ref with forwardRef.

Q20. What is React Fiber?

React’s reimplementation of the core algorithm. Enables incremental rendering, prioritization, and pausing work.

Q21. How to handle forms in React?

Controlled components: value bound to state. Or uncontrolled with useRef.

function Form() {
  const [email, setEmail] = useState('');
  return <input value={email} onChange={e => setEmail(e.target.value)} />;
}

Q22. What is lifting state up?

Moving state to the closest common ancestor so multiple components can share it.

Q23. What are portals?

Render children into a DOM node outside the parent component’s DOM hierarchy (e.g., modals).

ReactDOM.createPortal(<Modal />, document.getElementById('modal-root'));

Q24. What is the difference between useState and useReducer?

useState is simpler. useReducer is better for complex state transitions and when next state depends on previous.

Q25. How to debug performance issues?

Use React DevTools Profiler, Chrome Performance tab, why-did-you-render library.

Q26. What is code splitting?

Splitting bundle into smaller chunks loaded on demand. Reduces initial load time.

const LazyPage = React.lazy(() => import('./Page'));

Q27. What are React Hooks rules?

1. Only call hooks at the top level
2. Only call hooks from React functions
3. Don't call hooks inside loops, conditions, or nested functions

Q28. What is the Context API?

Provides a way to pass data through the component tree without prop drilling.

Q29. When should you not use index as key?

When list items can change order, be added/removed. Index causes incorrect reconciliation.

Q30. What is useDeferredValue?

Defers updating a value until more urgent updates are done (React 18).

Q31. What is useTransition?

Marks updates as non-urgent. Allows keeping UI responsive (React 18).

const [isPending, startTransition] = useTransition();
startTransition(() => {
  setTab('heavy');
});

Q32. How to optimize images in React?

Use <img loading="lazy">, WebP format, responsive images with srcset, or libraries like react-lazy-load-image-component.

Q33. What is Server Components (RSC)?

React 18+ feature (Next.js). Components rendered on server, no client JS. Reduces bundle size.

Q34. What is the difference between client and server components?

Client: Runs in browser, can use hooks/state. Server: Runs on server, no interactivity, faster initial load.

Q35. How to handle authentication in React?

Store JWT in httpOnly cookie or memory. Use protected routes. Refresh token flow.

Q36. What is React Query?

Library for managing server state: caching, refetching, pagination. Alternative to Redux for API data.

Q37. What is the use of useId hook?

Generates unique IDs for accessibility (labels, aria attributes). Works with SSR.

const id = useId();
<label htmlFor={id}>Name</label>
<input id={id} />

Q38. How to test React components?

Use Jest + React Testing Library. Test behavior, not implementation.

render(<Button />);
expect(screen.getByText('Click')).toBeInTheDocument();

Q39. What is the difference between controlled and uncontrolled components?

Controlled: Value managed by React state. Uncontrolled: Value managed by DOM (use ref).

Q40. How to deploy a React app?

Build with npm run build. Deploy build/ folder to Vercel, Netlify, AWS S3, or any static host.

Key Takeaways

  • Always cleanup in useEffect
  • Use React.memo, useMemo, useCallback wisely
  • Prefer functional components and hooks
  • Code split heavy components
  • Use stable keys in lists
  • Handle errors with Error Boundaries
  • Protect routes and manage auth state
  • Test behavior, not implementation
