Core Concepts

What is Node.js?

Node.js is a JavaScript runtime built on Chrome’s V8 engine. It allows you to run JS outside the browser, enabling server-side applications. It uses an event-driven, non-blocking I/O model.

How Node.js Works

Single-threaded: One main thread for JavaScript execution.
Event Loop: Manages async callbacks.
Non-blocking I/O: File system, network, etc., are handled by libuv (C library).

Modules: require & exports

// math.js
exports.add = (a, b) => a + b;

// app.js
const { add } = require('./math');
console.log(add(2, 3)); // 5

Asynchronous Programming

Callbacks

fs.readFile('file.txt', (err, data) => {
  if (err) return console.error(err);
  console.log(data.toString());
});

Promises

const readFile = util.promisify(fs.readFile);
readFile('file.txt')
  .then(data => console.log(data.toString()))
  .catch(err => console.error(err));

Async/Await

async function read() {
  try {
    const data = await fs.promises.readFile('file.txt', 'utf8');
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

Building a REST API (Native Node.js)

HTTP Server

const http = require('http');
const url = require('url');

const server = http.createServer((req, res) => {
  const parsedUrl = url.parse(req.url, true);
  const method = req.method;
  const path = parsedUrl.pathname;

  res.setHeader('Content-Type', 'application/json');

  if (method === 'GET' && path === '/api/users') {
    res.writeHead(200);
    res.end(JSON.stringify({ users: ['Alice', 'Bob'] }));
  } else if (method === 'POST' && path === '/api/users') {
    let body = '';
    req.on('data', chunk => body += chunk);
    req.on('end', () => {
      const user = JSON.parse(body);
      res.writeHead(201);
      res.end(JSON.stringify({ message: 'User created', user }));
    });
  } else {
    res.writeHead(404);
    res.end(JSON.stringify({ error: 'Not Found' }));
  }
});

server.listen(3000, () => console.log('Server running on port 3000'));

CRUD Routes

  • GET /users → List
  • GET /users/1 → Read
  • POST /users → Create
  • PUT /users/1 → Update
  • DELETE /users/1 → Delete

Advanced Topics

Buffers

Raw binary data. Used for file I/O, network streams.

const buf = Buffer.from('Hello');
console.log(buf); // <Buffer 48 65 6c 6c 6f>
console.log(buf.toString()); // Hello

Streams

Efficiently handle large data. Types: Readable, Writable, Duplex, Transform.

const readStream = fs.createReadStream('large.txt');
const writeStream = fs.createWriteStream('copy.txt');

readStream.pipe(writeStream);

Cluster Module

Fork multiple worker processes to use all CPU cores.

const cluster = require('cluster');
const os = require('os');

if (cluster.isMaster) {
  const cpus = os.cpus().length;
  for (let i = 0; i < cpus; i++) {
    cluster.fork();
  }
} else {
  http.createServer((req, res) => {
    res.end('Hello from worker');
  }).listen(3000);
}

40 Full Node.js Interview Questions & Answers

Q1. What is the Event Loop in Node.js?

The event loop is a core mechanism that allows Node.js to perform non-blocking I/O operations despite being single-threaded. It continuously checks the call stack and task queues (macrotasks and microtasks). When the call stack is empty, it processes callbacks from the queues.

Phases: timers → pending callbacks → idle, prepare → poll → check → close callbacks → microtasks (process.nextTick, Promise)

setTimeout(() => console.log('Timer'), 0);
Promise.resolve().then(() => console.log('Promise'));
console.log('Sync');
// Output: Sync → Promise → Timer

Q2. Is Node.js single-threaded?

Yes, the JavaScript execution is single-threaded, but I/O operations (file, network) are offloaded to the libuv thread pool (default 4 threads). This enables non-blocking behavior.

fs.readFile('file.txt', () => { /* callback */ }); // Offloaded to thread pool
console.log('Next line'); // Runs immediately

Q3. What is the difference between require and import?

require: CommonJS, synchronous, runtime loading.
import: ES6 modules, static, compile-time, supports top-level await.

// CommonJS
const fs = require('fs');

// ESM
import fs from 'fs';

Q4. Explain process.nextTick() vs setImmediate().

process.nextTick(): Executes in the microtask queue before any I/O or timers.
setImmediate(): Executes in the check phase of the event loop, after I/O.

process.nextTick(() => console.log('nextTick'));
setImmediate(() => console.log('setImmediate'));
// Output: nextTick → setImmediate

Q5. What are Streams in Node.js?

Streams are collections of data processed in chunks. Ideal for large files or real-time data. Types: Readable, Writable, Duplex, Transform.

const readStream = fs.createReadStream('bigfile.mp4');
readStream.pipe(res); // Stream video to HTTP response

Q6. What are Buffers?

Fixed-size chunks of memory for handling raw binary data. Used in file I/O, network streams, cryptography.

const buf = Buffer.alloc(10);
buf.write('Hello');
console.log(buf.toString()); // Hello

Q7. How to handle uncaught exceptions?

Use process.on('uncaughtException') and unhandledRejection. Log and gracefully shutdown.

process.on('uncaughtException', (err) => {
  console.error('Uncaught:', err);
  process.exit(1);
});

Q9. How to scale a Node.js app?

Use cluster module, PM2, Kubernetes, load balancer. Horizontal scaling with multiple instances.

Q10. What is libuv?

C library that handles asynchronous I/O. Provides thread pool, event loop, file system, DNS, etc.

Q11. What is __dirname and __filename?

Global variables in CommonJS. __dirname: current directory path. __filename: current file path.

console.log(__dirname);  // /home/user/project
console.log(__filename); // /home/user/project/server.js

Q12. How to read environment variables?

Use process.env or dotenv package.

require('dotenv').config();
const port = process.env.PORT || 3000;

Q13. What is middleware in Express?

Functions that have access to req, res, and next. Used for logging, auth, parsing.

Q14. How to handle file uploads in Node.js?

Use multer (Express) or parse multipart/form-data manually with busboy.

Q15. What is the difference between development and production mode?

Development: hot reload, detailed errors. Production: minified, no source maps, optimized.

if (process.env.NODE_ENV === 'production') {
  // Disable logs
}

Q16. How to prevent callback hell?

Use Promises, async/await, or modularize code.

async function main() {
  const user = await getUser();
  const posts = await getPosts(user.id);
}

Q17. What is zlib module?

Provides compression/decompression (gzip, deflate).

const zlib = require('zlib');
zlib.gzip(data, (err, compressed) => { ... });

Q18. What is REPL?

Read-Eval-Print Loop. Interactive Node.js shell.

$ node
> 2 + 2
4

Q19. How to debug Node.js apps?

Use --inspect flag, Chrome DevTools, VS Code debugger, or console.log.

node --inspect-brk server.js

Q20. What is pm2?

Process manager for Node.js. Features: clustering, monitoring, zero-downtime reload.

pm2 start app.js --name "api"
pm2 restart api
pm2 logs

Q21. What is worker_threads?

Allows true parallelism in Node.js using multiple threads.

const { Worker } = require('worker_threads');
const worker = new Worker('./worker.js');

Q22. How to secure a Node.js app?

Use HTTPS, helmet, rate limiting, input validation, JWT, CORS, environment variables.

Q23. What is npm ci?

Clean install from package-lock.json. Faster, deterministic. Used in CI/CD.

Q24. What is nodemon?

Tool that restarts the app when file changes are detected.

npm install -D nodemon
npx nodemon server.js

Q25. What is the difference between node and nodejs?

No difference. nodejs is the package name in some Linux distros.

Q26. How to handle large file uploads?

Use streams. Avoid loading entire file into memory.

req.pipe(fs.createWriteStream('upload.mp4'));

Q27. What is crypto module?

Provides cryptographic functionality: hashing, HMAC, cipher, decipher.

const hash = crypto.createHash('sha256').update('password').digest('hex');

Q28. What is dns module?

Performs DNS lookups and reverse lookups.

dns.lookup('google.com', (err, address) => {
  console.log(address);
});

Q29. What is child_process?

Allows spawning subprocesses: shell commands, other Node.js scripts.

Q30. How to implement rate limiting?

Use express-rate-limit or custom middleware with Redis.

Q31. What is global object in Node.js?

Similar to window in browser. Contains console, process, setTimeout, etc.

Q32. How to profile Node.js performance?

Use clinic.js, --prof flag, or Chrome CPU profiler.

Q33. What is V8 engine?

Google’s open-source JS engine. Compiles JS to machine code. Used in Chrome and Node.js.

Q34. What is heap and stack in Node.js?

Stack: Function call frames, primitives.
Heap: Objects, closures, dynamic memory.

Q35. How to handle CORS in Node.js?

Set Access-Control-Allow-Origin header or use cors middleware.

res.setHeader('Access-Control-Allow-Origin', '*');

Q36. What is middleware pattern?

Chain of functions that process request/response. Each calls next().

Q37. How to implement caching?

Use in-memory (Node cache), Redis, or HTTP caching headers.

Q38. What is graceful shutdown?

Close server, finish in-flight requests, close DB connections before exit.

process.on('SIGTERM', () => {
  server.close(() => {
    db.close();
    process.exit(0);
  });
});

Q39. What is npm shrinkwrap?

Deprecated. Use package-lock.json for deterministic installs.

Q40. How to deploy Node.js app?

Use PM2, Docker, Kubernetes, or platforms like Render, Railway, AWS EC2.

pm2 start ecosystem.config.js --env production

Key Takeaways

  • Event loop is the heart of Node.js
  • Use async/await over callbacks
  • Always handle errors and cleanup
  • Use streams for large data
  • Scale with cluster or PM2
  • Secure with HTTPS, JWT, rate limiting
  • Monitor with PM2, logs, health checks
  • Graceful shutdown is critical
